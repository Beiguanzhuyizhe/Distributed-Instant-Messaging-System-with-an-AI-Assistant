# P2P 文件传输实验说明

## 定位

本项目包含 P2P 辅助打洞相关代码：

- `server/p2p_helper.py`
- `client/p2p.py`
- 协议消息：`P2P_HOLE_PUNCH`、`P2P_READY`

当前最终演示建议以“服务端中继文件传输”为主，P2P 作为实验性扩展说明。

## 已实现内容

- 客户端可向服务端发送 P2P 打洞请求。
- 服务端维护在线用户地址信息。
- 服务端协助双方交换地址，并向目标用户发送通知。
- 服务端调用 `handle_hole_punch` 时会传入目标连接，地址交换链路完整。

## 限制

P2P UDP 打洞受以下条件影响：

- NAT 类型
- 防火墙策略
- 校园网/宿舍网隔离
- 本机安全软件
- 双方是否在同一局域网或可互达网络中

因此不能承诺在所有现场网络环境下成功。对称 NAT 或严格防火墙环境下，P2P 很可能失败。

## 演示建议

推荐演示路径：

1. 私聊/群聊确认通信正常。
2. 使用服务端中继发送文件，保证必成。
3. 展示协议文档和代码中 P2P 打洞扩展点。
4. 如果现场网络允许，再尝试 P2P 打洞；失败时说明自动/手动回到中继方案。
